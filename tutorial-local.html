tacitscript tutorial

tacitscript - a language for building algorithms

tacitscript is a programming language optimized for building algorithms from composable parts, through a simplified, concise syntax. To learn more about the ideas behind tacitscript, see here.

This tutorial will introduce the language incrementally through example and interactive exercises.

Lesson 1: tacitscript (ts) blocks

tacitscript blocks of code are embedded within normal JavaScript code.

They are defined within special comment blocks beginning with a /*ts and ending with a */. The tacitscript transpiler will convert these blocks to normal JavaScript code. Definitions are shared freely between js and ts code.

const jsVariable = 4; /*ts number 3 // this is a number decimal 3.5 also a number. NB: terminating line comments need not be prefixed with // string "strings may continue over several lines" they are delimited by double-quotes only falsey () there is only one falsey value, undefined, denoted () array (1 "hello" () number) arrays are bounded by brackets and delimited by any kind of whitespace expression 2*jsVariable js variables in scope may be directly referenced within ts block pitfall 4 + number NB: tokens cannot be freely separated by whitespace */ console.log(number); // 3 console.log(decimal); // 3.5 console.log(string); // "strings may continue\nover several lines" console.log(falsey); // undefined console.log(array); // [1, "hello", undefined, 3] console.log(expression); // 8 console.log(pitfall); // 4 ("+ number" is considered a comment)

  • Each ts block consists of a list of name-value bindings.
  • Each ts name is made from upper and lower-case alphabetic characters only.
  • The value bound to a name is given by any valid ts expression.
  • Any characters on the line following a closed ts expression are considered to be from a comment.
